# AGI元认知检测工具 — 产品规划

> 基于意识拓扑引擎Chern熵判定，做第一个可量化AGI元认知的benchmark。
> 民国工匠隐士的第89代：快到头了，把东西做出来。

---

## 五维度定位

| 维度 | 测什么 | 当前状态 |
|------|--------|----------|
| 元认知 | 不确定时知不知道闭嘴 | ✅ Chern熵判定已跑通 |
| 推理深度 | 能不能多步推理不跳步 | ❌ 待建 |
| 迁移泛化 | 训练场景A→测试场景B | ❌ 待建 |
| 自我修正 | 走错了能不能回头 | ❌ 待建 |
| 长期规划 | 局部贪心够不着时能不能规划 | ❌ 待建 |

---

## 取舍

做：
- 标准化评测API——接外部模型
- 多类不确定性场景——不只导航
- 可量化评分——四个指标归一化
- 可读测试报告——不是表格，是评估

不做：
- 不训模型——只测，不造
- 不搞排行榜——排名是鸦片，信噪比才是数据
- 不接GPU集群——本地单机跑得动就行
- 不包装成"AGI检测完整方案"——声明边界：五维度里只测元认知

## 架构

```
被测模型（任意）
    │
    ▼
评测API层 ─── predict(state) → action
    │
    ├── 场景1: 感知模糊（视觉噪声递增）
    ├── 场景2: 规则冲突（两条规则打架）
    ├── 场景3: 长期规划（局部贪心失效）
    └── 场景4: 网格导航（原benchmark保留）
    │
    ▼
Chern熵判定引擎
    │
    ├── 信噪比：低熵输出占比
    ├── 闭嘴率：高熵时退回安全策略的比例
    ├── 过度自信率：高熵时仍强行输出的比例
    └── 场景覆盖度：四场景加权综合
    │
    ▼
报告生成 → JSON + Markdown + 可视化
```

## 里程碑

### M1: 元认知维度跑稳
- 标准化模型接口 `predict(state) → action`
- 接入至少2个外部模型（OpenAI API + 本地MLP对比）
- 四指标评分体系上线
- 报告生成（JSON + Markdown）
- **目标**：别人clone下来能把自己的模型接进来跑

### M2: 多场景
- 感知模糊场景（噪声递增网格）
- 规则冲突场景（两个reward信号打架）
- 场景覆盖度加入评分
- **目标**：不只测导航，测元认知在多个不确定性类型上的表现

### M3: 推理+修正
- 5-10步因果链迷宫（必须按序解锁的门/钥匙）
- 错误路径+中途提示→修正率
- 完整推理率、修正成功率
- **目标**：从"元认知"扩展到"元认知+推理+修正"三维

### M4: 迁移+规划
- 跨分布迁移测试（障碍类型/密度/规则全换）
- 100x100长程规划场景
- 五维度全部上线
- Web操作界面
- **目标**：完整五维度AGI元认知检测工具，一站式

## M1第一步：拆Chern引擎 + 定义API

1. 从 `benchmark.py` 抽出 `ChernWalk` 类 → `chern_engine.py`
2. 定义模型接口 `api.py`：
   ```python
   class ModelInterface:
       def predict(self, state: dict) -> str:
           """返回 action，必须是 legal_actions 之一"""
           pass
   ```
3. 写第一个外部模型适配器 `openai_adapter.py`
4. 接入Chern引擎跑评测，输出四指标
